# Build Lab Collaborations

Active client projects in progress.

## Folder Structure

Each collaboration folder contains:

```
{project-id}-{project-name}/
  BRIEFING.md       # Original intake + requirements
  ITERATION_LOG.md  # Build progress and decisions
  STATUS.md         # Current status for client
  /feedback/        # Client feedback files
  /assets/          # Logos, images, content
```

## Starting a New Collaboration

1. Client submits intake form at buildlab.com/intake
2. API creates folder: `{project-id}-{slug}/`
3. BRIEFING.md populated from intake data
4. Client receives status page link
5. Build begins, ITERATION_LOG.md updated throughout
